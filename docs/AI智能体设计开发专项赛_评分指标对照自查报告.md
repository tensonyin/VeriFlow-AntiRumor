## 🔍 各指标技术实现与得分点详述

### 一、 创新性维度 (占比 25%)

#### 1. 主题有新意 (15分) —— *让裁判看到独特的产品定义*
*   **社会痛点精准**：针对“银发族”（老年群体）在 AIGC 时代容易受到“伪科学养生”、“恐吓营销”及“大模型幻觉”毒害的现状进行精准立项。
*   **解决方案新颖**：区别于传统单向的、晦涩的大段文字辟谣，本项目在后台使用**红蓝对抗事实核查机制**，在前端设计了**双模态一键切换**。不仅展现了“白盒化”的科技推导过程以赢得年轻用户的信任，更提供了面向老年人的极简、温馨、大字号“安心电台式”视听体验。

#### 2. 整体设计涉及多学科融合 (10分) —— *让裁判看到跨领域的综合能力*
*   **人工智能与数据处理（AI/RAG）**：设计多智能体级联管道，调用 Tavily 实现中英文实时跨境 RAG（检索增强生成）与逻辑分析。
*   **设计艺术与人机工学**：采用沙漠温馨暖色调；自适应 1.5 倍字体排版，并首创 LaTeX 红头养生大字报，符合传统报刊阅读直觉；引入热敏小票和盖章动效。
*   **社会心理学（等待焦虑控制）**：普通模式用“思维风暴”SSE 日志流播报满足求知欲，长辈模式用“绿色高可见度进度条”+ 打字音效安抚老人的空白等待焦虑。
*   **嵌入式与电子工程**：基于 Linux 固件编写行空板 (Unihiker) 自适应 Python 引导，实现一键物理按键录音求助的软硬一体系统。

### 三、 先进性维度 (占比 30%)

#### 1. 工作流开发 (10分) —— *让裁判看到强悍的决策大脑 (超10节点)*
*   本项目在 Dify 平台深度定制了多达 **23个节点** 的级联核查工作流，完全超越了初高中组“必须包含 10 个节点以上”的要求。
*   **包含多处智能决策判断（非线性执行）**：
    1.  **特征感知判定**：智能判定输入素材的类型（OCR、视频、音频），并提取伪造畸变特征。
    2.  **跨境检索决策**：根据流言涉及的背景，自主决策是否启动翻译引擎并在国际 Tavily Search 上追加英文 `hoax` / `debunk` 后缀发起二次跨境检索。
    3.  **充分性条件分支 (If-Else)**：校验正反两方报告是否均有有效内容，若输入为无意义字符则强制分流到“异常输出”节点拦截。
    4.  **大法官定性裁决 (Final Judge)**：对红蓝对抗矩阵进行最终裁决，得出“证实”、“伪造”或“存疑”的终判定。
    5.  **合规自愈修正**：自适应校验生成的报告中是否含有 404/失效链接，自动清洗死链，并在大模型侧无缝重写重构。
    6.  **用户模式路由**：读取前端全局变量 `isElderlyMode` 智能分流，普通模式输出专业小票，长辈模式则生成“有声安心广播稿”与 LaTeX 大字报。

#### 2. 多模态调用 (5分) —— *让裁判看到前沿大模型的感知力*
*   在 [Content Sensing 节点](file:///d:/Desktop/谣言终结者/dify_workflows/谣言终结者：基于多源异构对抗博弈的多模态事实核查系统%20(12).yml)中调用大语言模型开启视觉 High 模式，实现对传言图片、截图 OCR 文本、AI 合成图的线条畸变特征等进行多模态预处理。

#### 3. 多智能体协同 (5分) —— *让裁判看到 Multi-Agent 架构的先进性*
*   系统中构建了清晰的智能体角色矩阵：
    *   **取证探员 (Forensic Agent)**：专注于事实检索与链接提取。
    *   **审计探员 (Logic Judgment)**：死磕常识漏洞、物理违背与情绪恐吓。
    *   **首席大法官 (Final Judge)**：中立地整合对比两方意见，完成独立审计并下达定性判定。
    *   **合规专家 (Compliance Agent)**：驱动代码沙箱进行链接过滤与重写修正。

#### 4. 智能人机交互 (5分) —— *让裁判感受极致的用户体验*
*   **可视化思维风暴**：普通模式下，前端借助 Server-Sent Events (SSE) 流式接收后台状态，在 [ThinkingWorkflow.tsx](file:///d:/Desktop/谣言终结者/src/components/ThinkingWorkflow.tsx) 组件中以动态手风琴卡片形式还原 AI 探员的实时推理细节，展现“白盒化”特征。
*   **拖拽捏合推导画布**：前端 [MermaidChart.tsx](file:///d:/Desktop/谣言终结者/src/components/MermaidChart.tsx) 集成了 `d3-zoom` 交互机制，支持用户在 Web 端对逻辑推导图进行双指捏合缩放（Zoom）和自由拖拽（Drag），彻底推平了传统 Mermaid 拓扑图在长辈模式大字号下文字堆叠的顽疾。
*   **温情儿女声 TTS**：长辈模式输出专用的广播稿，可自由选择“儿子声”或“女儿声”的本地 Edge-TTS 语音合成进行温暖降速播报。

#### 5. 提示词设计质量 (5分) —— *让裁判看到精细的 Prompt 工程能力*
*   在 [BACKEND_PROMPTS.md](file:///d:/Desktop/谣言终结者/BACKEND_PROMPTS.md) 中完整定义了各个阶段的 Prompt，例如：
    *   **媒介感知合规红线**：严防大语言模型幻觉，限定没有多模态文件时绝对不能臆造物理特征。
    *   **LaTeX 汉字防堆叠规则**：针对前端 MathJax 的局限性，在 LaTeX 生成节点设定了在每一个汉字或中文标点间必须强制插入“`\ `”或“`．`”的转义指令，防止渲染成海报时汉字挤作一团。
    *   **真相法庭独立审计原则**：指导模型对 Tavily 检索出已反转的新闻、废止的旧规或过时的声明进行二次修正。

### 四、 扩展性维度 (占比 10%)

#### 1. 可以扩展到特定设备中 (5分) —— *让裁判看到软硬一体的落地应用*
*   **行空板硬件集成**：将事实核查核心拓展嵌入到了 DFRobot 行空板上，运行 [unihiker_app.py](file:///d:/Desktop/谣言终结者/unihiker_app.py)。
*   **高保真 PC 仿真模拟器**：在开发阶段，通过检测行空板 GUI 库环境，自适应在电脑端拉起 Tkinter 仿真模拟界面，映射物理 A/B 键及音频，成倍提升了软硬件联调效率。
*   **首创“远程虚拟扬声器”中转机制**：针对微型硬件不便携带扬声器的硬件限制，行空板通过网络广播将 Edge-TTS 音频数据推回已连通的 React Web 客户端，让高品质的电脑扬声器充当硬件端的虚拟喇叭，实现了软硬联动的声音分发。

#### 2. “输出校验”功能 (5分) —— *让裁判看到防止幻觉与系统崩溃的鲁棒性*
*   **内置 Python 代码可用性校验**：[Compliance Agent](file:///d:/Desktop/谣言终结者/dify_workflows/谣言终结者：基于多源异构对抗博弈的多模态事实核查系统%20(12).yml) 会自主生成并运行 Python 脚本并发测试报告内引用的 URL。通过 HTTP 状态码（非 200 即视为失效）进行严格校验，过滤死链后重构文本，实现“大模型输出 $\rightarrow$ 代码运行校验 $\rightarrow$ 大模型修正”的闭环，彻底根治死链幻觉。
*   **非 BMP 字符 (Emoji) 安全防崩溃装饰器**：行空板 Tkinter 底层引擎在绘制表情符号（大于 `0xffff`）时会致命闪退。我们在 [unihiker_app.py](file:///d:/Desktop/谣言终结者/unihiker_app.py#L329-L332) 中设计了装饰器，正则拦截并洗涤非 BMP 字符，保障硬件端 100% 坚韧稳定。
*   **Mermaid 标签与 LaTeX 语法校验**：在工作流中建立转义规则，过滤 `()` `[]` `""`，防止渲染引擎发生解析崩溃。
